* These have been introduced in the deep planning networks paper, as a component. The DePlaNet takes sequences of actions in the latent space. To be honest, I'm not going to cover everything here; just want to know how the RSSM helps.
* I'm trying to be a little bit more precise here: when they say planning, they mean to learn a good dynamics model that can learn how to predict the next steps.
  * Let's talk more about this. according to claude the planning stage is the stage-2 of the algorithm where the model does it's own rollouts and also determines which actions will yield the best cumulative reward.
  * It does this by going through the environment again, and this time it makes up k new steps, ingests them and outputs (observation, action, reward) triplets. The actions are determined by CEM though (which i'm not going to cover here).
* The environments for this paper are considered to be a POMDP (Partially observable markov decision processes) which defines (1) Transition function (2) Observation function (3) reward function (4) policy function.
  * Note about this: a partially observable MDPs are MDPs that don't get the full state as the input. In real world, apparently we can never almost give the full state as the input but only a partial, approximated version of it (for ex: an image of the current situation doesn't tell us about the composition of the image in terms of physics or chem).
* They use CEM/Cross entropy method for the planning stage to generate new data. Apparently there's this whole lineage of papers that starts with sutton stating that a mix of real world and synthetic examples are good for enhancing perf (sounds familiar)
